О проекте

Брокерское приложение — учебный пет-проект, имитирующий работу реального брокерского сервиса и торговлю ценными бумагами на бирже. Разработан с целью углублённого изучения микросервисной архитектуры, работы с Apache Kafka, распределённых транзакций и интеграции с внешними биржевыми API.

Приложение позволяет:

    регистрироваться и проходить аутентификацию,

    управлять валютными кошельками (пополнение, вывод),

    просматривать рынок ценных бумаг с актуальными ценами,

    формировать заявки на покупку/продажу,

    отслеживать состояние портфеля и историю транзакций.

Котировки обновляются в реальном времени через интеграцию с Finnhub API.
Архитектура и технологии

Backend построен на Java (Spring Boot) в виде 5 независимых микросервисов:

    gateway-service — единая точка входа, маршрутизация запросов,

    auth-service — регистрация, аутентификация, JWT-токены,

    market-service — данные о ценных бумагах и рыночные котировки,

    order-service — обработка заявок на покупку/продажу,

    savings-service — управление кошельками и портфелем пользователя.

Безопасность обеспечена Spring Security с JWT-аутентификацией.

Взаимодействие между сервисами реализовано через Apache Kafka (публикация событий, гарантии доставки, идемпотентная обработка сообщений).

Для каждой бизнес-сущности спроектированы отдельные PostgreSQL базы данных (схемы, хранимые процедуры, триггеры, индексы).

Вся инфраструктура (сервисы, Kafka, Zookeeper, базы данных) поднимается одной командой в Docker Compose.

Стек: Java, Spring, Hibernate, PostgreSQL, Apache Kafka, Docker, Finnhub API, HTML, CSS, JavaScript.

Быстрый старт
1. Клонируйте репозиторий

git clone https://github.com/your-username/broker-microservices.git
cd broker-microservices

2. Настройте переменные окружения (Опционально)

Откройте .env и укажите ваш реальный API-ключ Finnhub:

# Finnhub API Key (получить можно на https://finnhub.io/register)
FINNHUB_API_KEY=ваш_реальный_ключ_сюда

3. Убедитесь, что нужные порты свободны

Перед запуском проверьте, что следующие порты не заняты другими программами:

    7070 (Gateway)

    7071 (Auth)

    7072 (Savings)

    7073 (Market)

    7074 (Order)

    7075 (Kafka)

    15432 – 15435 (базы данных)

    2181 (Zookeeper)

Если какой-то порт занят, освободите его или измените маппинг в docker-compose.yml.
4. Запустите проект

docker compose up --build

При первом запуске будут созданы и инициализированы базы данных, подняты Kafka и все микросервисы.
5. Проверьте работу

Откройте в браузере: localhost:7070
