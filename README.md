# Брокерское приложение

Учебный пет-проект, имитирующий работу брокерского приложения и торговлю ценными бумагами на бирже.

Проект создан для углублённого изучения микросервисной архитектуры, Apache Kafka, распределённых транзакций и интеграции с внешними API. Он охватывает полный цикл биржевых операций: от регистрации пользователя до обмена активами с использованием реальных рыночных котировок.

---

## Возможности

- 🔐 Регистрация и аутентификация пользователей
- 💳 Пополнение и вывод средств с валютных кошельков
- 📈 Просмотр рынка ценных бумаг с актуальными котировками
- 💼 Просмотр инвестиционного портфеля
- 🛒 Создание заявок на покупку и продажу активов
- 🤝 Обмен денежными средствами и ценными бумагами между пользователями
- 📜 Просмотр истории транзакций

> Рыночные котировки обновляются в режиме реального времени через интеграцию с **Finnhub API**.

---

## Архитектура

Backend реализован на **Java Spring Boot** в виде пяти независимых микросервисов:

- `gateway-service`
- `auth-service`
- `market-service`
- `order-service`
- `savings-service`

Каждый сервис имеет собственную базу данных PostgreSQL.

Взаимодействие между сервисами построено двумя способами:

- **REST API** — для синхронных запросов и получения данных;
- **Apache Kafka** — для публикации и обработки событий, обеспечивающих асинхронное взаимодействие между сервисами.

---

## Технические особенности

### Микросервисная архитектура

- пять независимых сервисов;
- API Gateway;
- отдельная база данных для каждого сервиса.

### Безопасность

- Spring Security;
- JWT-аутентификация;
- разграничение доступа.

### Работа с базой данных

Для каждого сервиса спроектирована отдельная PostgreSQL БД.

Используются:

- триггеры;
- индексы;
- хранимые процедуры;
- ролевой доступ.

### Взаимодействие сервисов

Для обмена данными используются два подхода.

**REST API**

- синхронные запросы между микросервисами;
- получение данных в режиме request-response.

**Apache Kafka**

- публикация и обработка событий;
- асинхронное взаимодействие между сервисами;
- гарантии доставки сообщений;
- идемпотентная обработка сообщений в консьюмерах.

### Интеграция с биржей

Сервис рынка интегрирован с **Finnhub API**, который регулярно получает актуальные котировки и обновляет их в приложении.

### Контейнеризация

Вся инфраструктура запускается одной командой через Docker Compose.

Поднимаются:

- все микросервисы;
- Apache Kafka;
- Zookeeper;
- PostgreSQL-инстансы.

# Быстрый старт

## 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/broker-microservices.git
cd broker-microservices
```

---

## 2. Настройка Finnhub API *(необязательно)*

Откройте файл `.env` и укажите ваш API-ключ.

```env
# Получить API-ключ можно на:
# https://finnhub.io/register

FINNHUB_API_KEY=your_api_key
```

Если ключ не указан, сервис рыночных данных не сможет получать актуальные котировки.

---

## 3. Проверьте свободные порты

| Сервис | Порт |
|---------|-----:|
| Gateway | 7070 |
| Auth | 7071 |
| Savings | 7072 |
| Market | 7073 |
| Order | 7074 |
| Kafka | 7075 |
| PostgreSQL | 15432–15435 |
| Zookeeper | 2181 |

Если какой-либо порт занят, освободите его либо измените соответствующий маппинг в `docker-compose.yml`.

---

## 4. Запуск проекта

```bash
docker compose up --build
```

При первом запуске автоматически:

- создаются и инициализируются базы данных;
- запускаются PostgreSQL-инстансы;
- поднимаются Apache Kafka и Zookeeper;
- запускаются все микросервисы.

---

## 5. Откройте приложение

После успешного запуска приложение будет доступно по адресу:

```text
http://localhost:7070
```
